# Инструкция по обогащению информации о майнерах и размещению на странице продукта

## Анализ текущей ситуации

### Что есть на promminer.ru (эталонная страница)

На эталонной странице майнера (например, Bitmain Antminer Z15 Pro) отображается следующая информация:

1. **Основная информация:**
   - Название модели (например: "ASIC-майнер Bitmain Antminer Z15 Pro 840KSol/s")
   - Производитель (Bitmain, MicroBT, и т.д.) - **ОШИБКА: у нас может быть написано "Bitman" вместо "Bitmain"**
   - Изображение майнера

2. **Технические характеристики:**
   - Алгоритм майнинга (Equihash, SHA-256, Scrypt, и т.д.)
   - Хэшрейт (840 kSol/s, 860 TH/s, и т.д.)
   - Потребляемая мощность (2780 Вт, 3250W, и т.д.)
   - Энергоэффективность (3,31 Дж/kSol, и т.д.)
   - Размеры (245 x 132 x 290 мм)
   - Вес (5,9 кг)
   - Уровень шума (75 дБ)
   - Интерфейс (RJ45 Ethernet 10/100M)
   - Рабочая температура (5 – 40 °C)
   - Влажность (10 – 90%)
   - Тип охлаждения (Воздушное, Водяное, и т.д.)

3. **Дополнительная информация:**
   - Описание майнера
   - Информация о производителе
   - Рекомендации по размещению

### Что есть в нашей базе данных (MinerDetail)

В сущности `MinerDetail` есть следующие поля:
- ✅ `standardName` - стандартизированное название
- ✅ `manufacturer` - производитель
- ✅ `series` - серия майнера
- ✅ `hashrate` - хэшрейт
- ✅ `algorithm` - алгоритм майнинга
- ✅ `powerConsumption` - потребление энергии
- ✅ `coins` - добываемые монеты
- ✅ `powerSource` - источник питания
- ✅ `cooling` - тип охлаждения
- ✅ `operatingTemperature` - рабочая температура
- ✅ `dimensions` - размеры
- ✅ `noiseLevel` - уровень шума
- ✅ `description` - описание
- ✅ `features` - особенности оборудования
- ✅ `placementInfo` - рекомендации по размещению
- ✅ `producerInfo` - информация о производителе

### Что отображается на нашей странице (http://localhost:8080/products/8)

На текущей странице отображается только:
- Название товара (из `product.model`, а не `minerDetail.standardName`)
- Изображение
- Количество предложений
- Описание (из `product.description`, а не `minerDetail.description`)
- Цена от минимального предложения
- Базовая информация (модель, количество предложений, дата создания)

**НЕ отображается:**
- ❌ Производитель (`minerDetail.manufacturer`)
- ❌ Серия (`minerDetail.series`)
- ❌ Алгоритм (`minerDetail.algorithm`)
- ❌ Хэшрейт (`minerDetail.hashrate`)
- ❌ Потребление энергии (`minerDetail.powerConsumption`)
- ❌ Добываемые монеты (`minerDetail.coins`)
- ❌ Размеры (`minerDetail.dimensions`)
- ❌ Вес (НЕ ХРАНИТСЯ в базе)
- ❌ Уровень шума (`minerDetail.noiseLevel`)
- ❌ Рабочая температура (`minerDetail.operatingTemperature`)
- ❌ Информация о производителе (`minerDetail.producerInfo`)
- ❌ Особенности оборудования (`minerDetail.features`)
- ❌ Рекомендации по размещению (`minerDetail.placementInfo`)

## Инструкция по обогащению информации

### Шаг 1: Проверка и исправление производителя

**Проблема:** Возможна ошибка в написании производителя (например, "Bitman" вместо "Bitmain").

**Действие:**
1. Открыть страницу майнера в админ-панели или через базу данных
2. Проверить поле `manufacturer` в таблице `miner_details`
3. Исправить ошибки в написании:
   - "Bitman" → "Bitmain"
   - "MicroBt" → "MicroBT"
   - И т.д.

**SQL запрос для проверки:**
```sql
SELECT DISTINCT manufacturer FROM miner_details ORDER BY manufacturer;
```

### Шаг 2: Сбор технических характеристик для каждой модели

Для каждой модели майнера необходимо собрать информацию из надежных источников:

**Источники информации:**
1. Официальный сайт производителя (Bitmain, MicroBT, и т.д.)
2. Проверенные интернет-магазины (promminer.ru, asicminervalue.com, и т.д.)
3. Специализированные ресурсы по майнингу

**Список характеристик для сбора:**
1. **Производитель** (manufacturer) - обязательно
2. **Модель** (standardName) - стандартизированное название
3. **Серия** (series) - например: "S21", "L7", "Z15"
4. **Алгоритм** (algorithm) - SHA-256, Scrypt, Equihash, и т.д.
5. **Хэшрейт** (hashrate) - например: "860 TH/s", "840 kSol/s"
6. **Потребление энергии** (powerConsumption) - например: "3250W", "2780 Вт"
7. **Энергоэффективность** - **НЕ ХРАНИТСЯ** (можно добавить в описание или в новое поле)
8. **Размеры** (dimensions) - например: "900 x 486.2 x 132 мм"
9. **Вес** - **НЕ ХРАНИТСЯ** (можно добавить в описание или в новое поле)
10. **Уровень шума** (noiseLevel) - например: "75 дБ", "40dB"
11. **Рабочая температура** (operatingTemperature) - например: "5 – 40 °C"
12. **Влажность** - **НЕ ХРАНИТСЯ** (можно добавить в описание или в новое поле)
13. **Интерфейс** - **НЕ ХРАНИТСЯ** (можно добавить в описание или в новое поле)
14. **Тип охлаждения** (cooling) - Воздушное, Водяное, Гибридное
15. **Добываемые монеты** (coins) - JSON массив или текст
16. **Описание** (description) - подробное описание майнера
17. **Особенности** (features) - ключевые особенности оборудования
18. **Информация о производителе** (producerInfo) - краткая информация о производителе
19. **Рекомендации по размещению** (placementInfo) - рекомендации по размещению и эксплуатации

### Шаг 3: Заполнение данных в MinerDetail

**Способ 1: Через админ-панель (если есть)**
1. Открыть страницу редактирования `MinerDetail`
2. Заполнить все доступные поля
3. Сохранить изменения

**Способ 2: Через базу данных (SQL)**
```sql
UPDATE miner_details 
SET 
    manufacturer = 'Bitmain',
    series = 'Z15',
    algorithm = 'Equihash',
    hashrate = '840 kSol/s',
    power_consumption = '2780 Вт',
    dimensions = '245 x 132 x 290 мм',
    noise_level = '75 дБ',
    operating_temperature = '5 – 40 °C',
    cooling = 'Воздушное',
    coins = '["ZEC", "ZEN"]',
    description = 'ASIC-майнер Bitmain Antminer Z15 Pro 840KSol/s для майнинга Zcash и других монет на алгоритме Equihash.',
    features = 'Высокая энергоэффективность, стабильная работа, низкий уровень шума.',
    producer_info = 'Bitmain - ведущий производитель ASIC-майнеров для майнинга криптовалют.',
    placement_info = 'Рекомендуется размещение в хорошо вентилируемом помещении при температуре 5-40°C.'
WHERE id = <ID_МАЙНЕРА>;
```

**Способ 3: Через API (если есть эндпоинт для редактирования)**
```bash
curl -X PUT http://localhost:8080/api/miner-details/{id} \
  -H "Content-Type: application/json" \
  -d '{
    "manufacturer": "Bitmain",
    "series": "Z15",
    "algorithm": "Equihash",
    "hashrate": "840 kSol/s",
    ...
  }'
```

### Шаг 4: Расширение сущности MinerDetail (опционально)

Если требуется хранить дополнительные поля, которых нет в текущей структуре:

**Поля, которые можно добавить:**
- `weight` (VARCHAR) - вес майнера в кг
- `energyEfficiency` (VARCHAR) - энергоэффективность (Дж/kSol, Дж/TH, и т.д.)
- `humidity` (VARCHAR) - допустимая влажность
- `interface` (VARCHAR) - интерфейс подключения (RJ45 Ethernet, и т.д.)

**Миграция:**
```sql
ALTER TABLE miner_details 
ADD COLUMN weight VARCHAR(50),
ADD COLUMN energy_efficiency VARCHAR(50),
ADD COLUMN humidity VARCHAR(50),
ADD COLUMN interface VARCHAR(100);
```
