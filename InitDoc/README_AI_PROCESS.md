# Процесс заполнения данных MinerDetail с помощью нейросети

## Обзор

Данная директория содержит документы для процесса заполнения технических характеристик и описаний для 46 устройств в таблице `miner_details` с помощью нейросети.

## Файлы в директории

1. **AI_PROMPT_FOR_MINER_DETAILS.md** - Промпт для нейросети с подробными инструкциями
2. **DATA_FORMAT_FOR_UPDATE.md** - Описание формата данных для обновления записей
3. **miners_list_for_ai.json** - Список всех 46 устройств в формате JSON
4. **README_AI_PROCESS.md** - Этот файл (инструкция по процессу)

## Процесс работы

### Шаг 1: Подготовка промпта

1. Открой файл `AI_PROMPT_FOR_MINER_DETAILS.md`
2. Скопируй содержимое промпта
3. Вставь в нейросеть (ChatGPT, Claude, Gemini и т.д.)

### Шаг 2: Получение данных от нейросети

1. Нейросеть обработает список из 46 устройств
2. Для каждого устройства заполнит все необходимые поля
3. Вернет данные в формате JSON массива

### Шаг 3: Сохранение результатов

1. Сохрани результат от нейросети в файл `miner_details_update.json`
2. Убедись, что JSON валидный (проверь через JSON validator)
3. Сохрани файл в директорию `InitDoc/`

### Шаг 4: Обновление записей в базе данных

Данные из JSON файла можно обновить одним из способов:

#### Вариант 1: Через API endpoint (рекомендуется)
Создать endpoint `/api/init/miner-details/batch-update`, который принимает JSON массив и обновляет все записи.

#### Вариант 2: Через индивидуальные запросы
Использовать существующий endpoint `/api/init/miner-detail/{id}/update` для каждой записи отдельно.

#### Вариант 3: Прямое обновление через код
Создать скрипт или использовать `InitController` для массового обновления.

### Шаг 5: Проверка результатов

1. Вызови endpoint `/api/init/miner-details/analyze-all` для анализа всех записей
2. Проверь статистику заполненности полей
3. Убедись, что все записи обновлены корректно

## Поля для заполнения

### Обязательные поля (2):
1. **id** (Long) - ID записи в базе данных
2. **standardName** (String, до 200 символов) - Название устройства

### Опциональные технические поля (11):
3. **manufacturer** (String, до 100 символов) - Производитель
4. **series** (String, до 100 символов) - Серия
5. **hashrate** (String, до 50 символов) - Хэшрейт
6. **algorithm** (String, до 50 символов) - Алгоритм майнинга
7. **powerConsumption** (String, до 50 символов) - Потребляемая мощность
8. **coins** (TEXT) - Добываемые монеты
9. **powerSource** (String, до 100 символов) - Тип источника питания
10. **cooling** (String, до 100 символов) - Тип охлаждения
11. **operatingTemperature** (String, до 100 символов) - Рабочая температура
12. **dimensions** (String, до 200 символов) - Размеры
13. **noiseLevel** (String, до 50 символов) - Уровень шума

### Опциональные текстовые поля (4):
14. **description** (TEXT) - Основное описание (200-300 слов)
15. **features** (TEXT) - Ключевые особенности (100-150 слов)
16. **placementInfo** (TEXT) - Рекомендации по размещению (100-150 слов)
17. **producerInfo** (TEXT) - Информация о производителе (100-150 слов)

## Важные замечания

1. **Кодировка**: Все файлы должны быть в кодировке **UTF-8**
2. **Валидность JSON**: JSON должен быть валидным
3. **Достоверность данных**: Используй только официальные спецификации производителей
4. **Единообразие**: Соблюдай единый формат для всех полей одного типа
5. **Актуальность**: Используй актуальные данные (2023-2024 годы)
