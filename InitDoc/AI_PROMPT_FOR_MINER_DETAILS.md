# Промпт для нейросети: Заполнение данных о майнерах

3. **Полнота**: Заполни все указанные поля для каждого устройства
4. **Качество описаний**: Описания должны быть информативными, понятными и профессиональными

## Формат вывода данных

Для каждого устройства верни данные в формате **JSON** с следующей структурой:

```json
{
  "id": 1,
  "standardName": "Antminer S19j PRO",
  "manufacturer": "Bitmain",
  "series": "S19j",
  "hashrate": "104 TH/s",
  "algorithm": "SHA-256",
  "powerConsumption": "3054W",
  "coins": "BTC, BCH, BSV",
  "powerSource": "Интегрированный",
  "cooling": "Воздушное",
  "operatingTemperature": "от 0 до 40°C",
  "dimensions": "370 x 195.5 x 290 мм",
  "noiseLevel": "75 дБ",
  "description": "Подробное описание устройства (200-300 слов)...",
  "features": "Список ключевых особенностей устройства (100-150 слов)...",
  "placementInfo": "Рекомендации по размещению и эксплуатации (100-150 слов)...",
  "producerInfo": "Информация о производителе и этой конкретной модели (100-150 слов)..."
}
```

### Важные замечания по полям:

#### standardName (String, до 200 символов)
- Точное название модели устройства
- Пример: "Antminer S19j PRO", "Whatsminer M30S++"

#### manufacturer (String, до 100 символов)
- Название производителя
- Пример: "Bitmain", "MicroBT", "Canaan", "Goldshell"

#### series (String, до 100 символов)
- Серия устройства
- Пример: "S19j", "L7", "M30S", "Z15"

#### hashrate (String, до 50 символов)
- Хэшрейт устройства с единицей измерения
- **Формат**: "[число] [единица]/s"
- **Единицы измерения**: 
  - Для SHA-256: "TH/s" (терахеш в секунду) или "EH/s" (эксахеш в секунду)
  - Для Scrypt: "MH/s" (мегахеш в секунду) или "GH/s" (гигахеш в секунду)
  - Для Equihash: "kSol/s" (килосолюций в секунду) или "MSol/s" (мегасолюций в секунду)
  - Для X11, X13 и др.: "MH/s", "GH/s"
- Примеры: "104 TH/s", "840 kSol/s", "9500 MH/s"

#### algorithm (String, до 50 символов)
- Алгоритм майнинга
- Примеры: "SHA-256", "Scrypt", "Equihash", "X11", "Blake2b", "Eaglesong"

#### powerConsumption (String, до 50 символов)
- Потребляемая мощность
- **Формат**: "[число]W" или "[число] Вт"
- Примеры: "3054W", "3472W", "2780W"
- **Важно**: Указывай максимальную потребляемую мощность согласно спецификации

#### coins (TEXT)
- Добываемые монеты (через запятую или слеш)
- **Формат**: "MONETA1, MONETA2, MONETA3" или "MONETA1/MONETA2/MONETA3"
- Примеры: 
  - Для SHA-256: "BTC, BCH, BSV"
  - Для Scrypt: "LTC, DOGE"
  - Для Equihash: "ZEC, ZEN"
  - Для Blake2b: "SIA"
  - Для X11: "DASH"

#### powerSource (String, до 100 символов)
- Тип источника питания
- Варианты: "Интегрированный", "Внешний"
- **Интегрированный** - блок питания встроен в устройство
- **Внешний** - требуется отдельный блок питания

#### cooling (String, до 100 символов)
- Тип охлаждения
- Варианты: "Воздушное", "Водяное", "Гибридное"
- Для большинства ASIC-майнеров: "Воздушное"

#### operatingTemperature (String, до 100 символов)
- Рабочая температура
- **Формат**: "от [мин] до [макс]°C" или "[мин]-[макс]°C"
- Примеры: "от 0 до 40°C", "5-35°C"

#### dimensions (String, до 200 символов)
- Габаритные размеры устройства
- **Формат**: "[длина] x [ширина] x [высота] мм" или "[длина] мм (Д) x [ширина] мм (Ш) x [высота] мм (В)"
- Примеры: "370 x 195.5 x 290 мм", "428 мм (Д) x 195 мм (Ш) x 290 мм (В)"

#### noiseLevel (String, до 50 символов)
- Уровень шума
- **Формат**: "[число] дБ" или "~[число] дБ"
- Примеры: "75 дБ", "~75 дБ"
- **Важно**: Указывай уровень шума на расстоянии 1 метр от устройства

#### description (TEXT)
- **Основное описание устройства** (200-300 слов)
- Должно содержать:
  - Краткую информацию о модели и производителе
  - Основное назначение устройства
  - Ключевые технические характеристики (хэшрейт, алгоритм, потребление)
  - Область применения и преимущества
- **Стиль**: Профессиональный, информативный, понятный для потенциальных покупателей
- **Язык**: Русский

#### features (TEXT)
- **Ключевые особенности и функции** (100-150 слов)
- Должно содержать:
  - Технологические особенности устройства
  - Преимущества перед конкурентами
  - Особенности конструкции и эксплуатации
  - Дополнительные функции (если есть)
- **Стиль**: Краткий, структурированный список особенностей
- **Язык**: Русский

#### placementInfo (TEXT)
- **Рекомендации по размещению и эксплуатации** (100-150 слов)
- Должно содержать:
  - Требования к помещению (вентиляция, температура, влажность)
  - Рекомендации по размещению (майнинг-фермы, домашние условия)
  - Особенности эксплуатации
  - Требования к инфраструктуре (электроснабжение, охлаждение)
- **Стиль**: Практический, информативный
- **Язык**: Русский

#### producerInfo (TEXT)
- **Информация о производителе и модели** (100-150 слов)
- Должно содержать:
  - Краткую информацию о производителе
  - История и достижения производителя
  - Информация о конкретной модели в линейке производителя
  - Позиционирование модели на рынке
- **Стиль**: Маркетинговый, но достоверный
- **Язык**: Русский

## Важные правила:

1. **Достоверность**: Используй только официальные спецификации производителей. Если данные недоступны или неясны - указывай это явно.

2. **Единообразие**: Соблюдай единый формат для всех полей одного типа (например, все размеры в мм, все мощности в W).

3. **Актуальность**: Используй последние официальные данные (2023-2024 годы).

4. **Полнота**: Заполни ВСЕ поля для каждого устройства. Если какое-то поле неизвестно - укажи "Не указано" или оставь пустым строкой "".

5. **Качество текстов**: 
   - Описания должны быть уникальными для каждой модели
   - Избегай шаблонных фраз
   - Используй профессиональную терминологию
   - Пиши для целевой аудитории (покупатели майнингового оборудования)

6. **Проверка данных**:
   - Хэшрейт должен соответствовать официальным спецификациям
   - Потребляемая мощность должна быть реальной для данного хэшрейта
   - Алгоритм должен соответствовать монетам
   - Размеры должны быть достоверными
