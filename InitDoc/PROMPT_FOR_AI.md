# Промпт для нейросети: Заполнение данных о майнерах

## Задача
Необходимо заполнить подробные, качественные и достоверные технические характеристики и описания для 46 ASIC-майнеров криптовалют. Данные будут использоваться для отображения на сайте интернет-магазина майнингового оборудования.

## Инструкция

Ты - эксперт по ASIC-майнерам для добычи криптовалют. Твоя задача - собрать и структурировать **достоверную, актуальную информацию** о каждом устройстве из списка ниже, используя только **официальные источники** и **публичные спецификации** производителей.

### Требования к данным:
1. **Точность**: Все технические характеристики должны соответствовать официальным спецификациям производителя
2. **Актуальность**: Используй только актуальные данные (2023-2024 года)
3. **Полнота**: Заполни все указанные поля для каждого устройства
4. **Качество описаний**: Описания должны быть информативными, понятными и профессиональными

## Подробное описание полей

### 1. id (Long, обязательное)
- **Тип**: Число
- **Описание**: ID записи в базе данных (указан в списке устройств выше)
- **Пример**: `1`, `8`, `46`

### 2. standardName (String, до 200 символов, обязательное)
- **Тип**: Строка
- **Описание**: Точное название модели устройства
- **Пример**: `"Antminer S19j PRO"`, `"Whatsminer M30S++"`
- **Формат**: Произвольная строка с названием модели

### 3. manufacturer (String, до 100 символов)
- **Тип**: Строка
- **Описание**: Название производителя
- **Пример**: `"Bitmain"`, `"MicroBT"`, `"Canaan"`, `"Goldshell"`, `"Jasminer"`, `"ElphapeX"`

### 4. series (String, до 100 символов)
- **Тип**: Строка
- **Описание**: Серия устройства
- **Пример**: `"S19j"`, `"L7"`, `"M30S"`, `"Z15"`
- **Формат**: Название серии без полного названия модели

### 5. hashrate (String, до 50 символов)
- **Тип**: Строка
- **Описание**: Хэшрейт устройства с единицей измерения
- **Формат**: `"[число] [единица]/s"`
- **Единицы измерения**:
  - Для SHA-256: `TH/s` (терахеш), `EH/s` (эксахеш)
  - Для Scrypt: `MH/s` (мегахеш), `GH/s` (гигахеш)
  - Для Equihash: `kSol/s` (килосолюций), `MSol/s` (мегасолюций)
  - Для X11, X13 и др.: `MH/s`, `GH/s`
- **Примеры**: `"104 TH/s"`, `"840 kSol/s"`, `"9500 MH/s"`

### 6. algorithm (String, до 50 символов)
- **Тип**: Строка
- **Описание**: Алгоритм майнинга
- **Примеры**: `"SHA-256"`, `"Scrypt"`, `"Equihash"`, `"X11"`, `"Blake2b"`, `"Eaglesong"`

### 7. powerConsumption (String, до 50 символов)
- **Тип**: Строка
- **Описание**: Потребляемая мощность
- **Формат**: `"[число]W"` или `"[число] Вт"`
- **Примеры**: `"3054W"`, `"3472W"`, `"2780W"`
- **Важно**: Указывай максимальную потребляемую мощность согласно официальной спецификации

### 8. coins (TEXT)
- **Тип**: Текст
- **Описание**: Добываемые монеты
- **Формат**: Список монет через запятую или слеш
- **Примеры**: 
  - Для SHA-256: `"BTC, BCH, BSV"`
  - Для Scrypt: `"LTC, DOGE"`
  - Для Equihash: `"ZEC, ZEN"`
  - Для Blake2b: `"SIA"`
  - Для X11: `"DASH"`

### 9. powerSource (String, до 100 символов)
- **Тип**: Строка
- **Описание**: Тип источника питания
- **Варианты**: 
  - `"Интегрированный"` - блок питания встроен в устройство
  - `"Внешний"` - требуется отдельный блок питания
- **Пример**: `"Интегрированный"`

### 10. cooling (String, до 100 символов)
- **Тип**: Строка
- **Описание**: Тип охлаждения
- **Варианты**: 
  - `"Воздушное"` - воздушное охлаждение (для большинства ASIC-майнеров)
  - `"Водяное"` - водяное охлаждение
  - `"Гибридное"` - комбинированное охлаждение
- **Пример**: `"Воздушное"`

### 11. operatingTemperature (String, до 100 символов)
- **Тип**: Строка
- **Описание**: Рабочая температура
- **Формат**: `"от [мин] до [макс]°C"` или `"[мин]-[макс]°C"`
- **Примеры**: `"от 0 до 40°C"`, `"5-35°C"`

### 12. dimensions (String, до 200 символов)
- **Тип**: Строка
- **Описание**: Габаритные размеры устройства
- **Формат**: `"[длина] x [ширина] x [высота] мм"` или `"[длина] мм (Д) x [ширина] мм (Ш) x [высота] мм (В)"`
- **Примеры**: `"370 x 195.5 x 290 мм"`, `"428 мм (Д) x 195 мм (Ш) x 290 мм (В)"`
- **Важно**: Все размеры в миллиметрах (мм)

### 13. noiseLevel (String, до 50 символов)
- **Тип**: Строка
- **Описание**: Уровень шума
- **Формат**: `"[число] дБ"` или `"~[число] дБ"`
- **Примеры**: `"75 дБ"`, `"~75 дБ"`
- **Важно**: Уровень шума на расстоянии 1 метр от устройства

### 14. description (TEXT, 200-300 слов)
- **Тип**: Текст
- **Описание**: Основное описание устройства
- **Содержание**:
  - Краткая информация о модели и производителе
  - Основное назначение устройства
  - Ключевые технические характеристики (хэшрейт, алгоритм, потребление)
  - Область применения и преимущества
- **Стиль**: Профессиональный, информативный, понятный для потенциальных покупателей
- **Язык**: Русский

### 15. features (TEXT, 100-150 слов)
- **Тип**: Текст
- **Описание**: Ключевые особенности и функции
- **Содержание**:
  - Технологические особенности устройства
  - Преимущества перед конкурентами
  - Особенности конструкции и эксплуатации
  - Дополнительные функции (если есть)
- **Стиль**: Краткий, структурированный список особенностей
- **Язык**: Русский

### 16. placementInfo (TEXT, 100-150 слов)
- **Тип**: Текст
- **Описание**: Рекомендации по размещению и эксплуатации
- **Содержание**:
  - Требования к помещению (вентиляция, температура, влажность)
  - Рекомендации по размещению (майнинг-фермы, домашние условия)
  - Особенности эксплуатации
  - Требования к инфраструктуре (электроснабжение, охлаждение)
- **Стиль**: Практический, информативный
- **Язык**: Русский

### 17. producerInfo (TEXT, 100-150 слов)
- **Тип**: Текст
- **Описание**: Информация о производителе и модели
- **Содержание**:
  - Краткая информация о производителе
  - История и достижения производителя
  - Информация о конкретной модели в линейке производителя
  - Позиционирование модели на рынке
- **Стиль**: Маркетинговый, но достоверный
- **Язык**: Русский

## Важные правила

1. **Достоверность**: Используй только официальные спецификации производителей. Если данные недоступны или неясны - указывай это явно или оставляй поле пустым.

2. **Единообразие**: Соблюдай единый формат для всех полей одного типа (например, все размеры в мм, все мощности в W).

3. **Актуальность**: Используй последние официальные данные (2023-2024 годы).

4. **Полнота**: Заполни ВСЕ поля для каждого устройства. Если какое-то поле неизвестно - укажи пустую строку `""`.

5. **Качество текстов**: 
   - Описания должны быть уникальными для каждой модели
   - Избегай шаблонных фраз
   - Используй профессиональную терминологию
   - Пиши для целевой аудитории (покупатели майнингового оборудования)

6. **Проверка данных**:
   - Хэшрейт должен соответствовать официальным спецификациям
   - Потребляемая мощность должна быть реальной для данного хэшрейта
   - Алгоритм должен соответствовать монетам
   - Размеры должны быть достоверными
