# Исправление ошибки сборки "файл занят другим процессом"

## Проблема
Ошибка: `FileSystemException: Процесс не может получить доступ к файлу, так как этот файл занят другим процессом`

## Причина
Spring Boot приложение все еще запущено и держит файлы в директории `target/classes` открытыми.

## Решение

### Вариант 1: Через IDE (рекомендуется)
1. **Остановите Spring Boot приложение** в вашей IDE (IntelliJ IDEA, Eclipse и т.д.)
   - Нажмите красную кнопку "Stop" на панели запуска
   - Или закройте окно запуска приложения

2. **Выполните очистку и сборку**:
   ```bash
   cd shop-backend
   mvn clean compile
   ```

### Вариант 2: Через диспетчер задач Windows
1. Откройте **Диспетчер задач** (Ctrl+Shift+Esc)
2. Найдите процессы:
   - `java.exe`
   - `javaw.exe`
   - Любые процессы с названием, содержащим "spring" или "shop-backend"
3. Завершите эти процессы (правый клик → Завершить задачу)

4. **Выполните очистку и сборку**:
   ```bash
   cd shop-backend
   mvn clean compile
   ```

### Вариант 3: Ручная очистка
1. **Закройте IDE** (IntelliJ IDEA, Eclipse и т.д.)
2. **Удалите директорию target вручную**:
   - Откройте проводник Windows
   - Перейдите в `C:\IdeaPRG\miners\shop-backend\`
   - Удалите папку `target`
3. **Откройте IDE заново** и выполните сборку

## После исправления
После успешной очистки, запустите приложение заново:
```bash
cd shop-backend
mvn spring-boot:run
```
